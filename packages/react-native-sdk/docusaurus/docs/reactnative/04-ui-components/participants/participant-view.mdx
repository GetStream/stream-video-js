[//]: # ()
[//]: # (The `ParticipantView` component is a core component that renders a participant's video and plays the participant's audio.)

[//]: # ()
[//]: # (It can toggle between video and avatar based on the participant's video state. It displays information about the participant, their device settings and connection quality. It also provides action buttons &#40;for example, to pin the participant&#41;.)

[//]: # ()
[//]: # (The `ParticipantView` component is used to build up the [different call layouts]&#40;../../../ui-components/core/call-layout&#41;.)

[//]: # ()
[//]: # (To optimize performance,)

[//]: # ()
[//]: # (- the component won't stream video for participants that are not visible in the viewport)

[//]: # (- for visible participants it will request the appropriate video track quality and resolution &#40;dynascale&#41;)

[//]: # ()
[//]: # (![Preview of the Call controls component.]&#40;../../assets/03-ui-components/participant.png&#41;)

[//]: # ()
[//]: # (:::note)

[//]: # ()
[//]: # (Please note that we suggest not to create a custom `ParticipantView` component as it contains complex, low-level logic related to performance. Please refer to the [Customization section]&#40;#customization&#41; to see how you can tailor the component to your needs.)

[//]: # ()
[//]: # (:::)

[//]: # (## Props)

[//]: # ()
[//]: # (### `participant`)

[//]: # ()
[//]: # (| Type                                                                                                            |)

[//]: # (| --------------------------------------------------------------------------------------------------------------- |)

[//]: # (| [`StreamVideoParticipant`]&#40;https://github.com/GetStream/stream-video-js/blob/main/packages/client/src/types.ts&#41; |)

[//]: # ()
[//]: # (The participant whose video/audio stream we want to play.)

[//]: # ()
[//]: # (### `ParticipantViewUI`)

[//]: # ()
[//]: # (| Type                                        |)

[//]: # (| ------------------------------------------- |)

[//]: # (| `ComponentType` \| `ReactElement` \| `null` |)

[//]: # ()
[//]: # (Override the default UI for rendering participant information/actions.)

[//]: # ()
[//]: # (### `VideoPlaceholder`)

[//]: # ()
[//]: # (| Type                                        |)

[//]: # (| ------------------------------------------- |)

[//]: # (| `ComponentType` \| `ReactElement` \| `null` |)

[//]: # ()
[//]: # (Override the default UI that's visible when a participant turned off their video.)

[//]: # ()
[//]: # (### `videoMode`)

[//]: # ()
[//]: # (| Type                          |)

[//]: # (| ----------------------------- |)

[//]: # (| `video` \| `screen` \| 'none' |)

[//]: # ()
[//]: # (The kind of video stream to play for the given participant. The default value is `video`. You can use `none` if you're building an audio-only call.)

[//]: # ()
[//]: # (### `className`)

[//]: # ()
[//]: # (| Type                    |)

[//]: # (| ----------------------- |)

[//]: # (| `string` \| `undefined` |)

[//]: # ()
[//]: # (Custom class applied to the root DOM element.)

[//]: # ()
[//]: # (### `refs`)

[//]: # ()
[//]: # (| Type                                                                               |)

[//]: # (| ---------------------------------------------------------------------------------- |)

[//]: # ()
[//]: # (An object with set functions meant for exposing the video and video placeholder elements to the integrators.)

[//]: # ()
[//]: # (### `muteAudio`)

[//]: # ()
[//]: # (| Type      |)

[//]: # (| --------- |)

[//]: # (| `boolean` |)

[//]: # ()
[//]: # (This prop is only useful for advanced use-cases &#40;for example building your own paginated layout&#41;. When set to `true` it will mute the given participant's audio stream on the client side. The local participant is always muted.)
